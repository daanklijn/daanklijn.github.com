I'm currently in the process of designing a new digital drum synthesizer. One of its requirements is that it needs to be capable of pitch shifting certain drum samples. This blog post explores one of the simpler pitch shifting algorithms, and tries to find out whether it meets the needs of my synthesizer.

We can decrease the pitch of this sample by simply stretching it over a longer time period. Or we can increase its pitch by compressing it into a shorter time period.

While this does work, there is one unwanted side-effect:
the sample's length changes as well.

This can be side-effect can be avoid by using proper pitch shifting methods. One of the simpler ones is Overlap Add (OLA).

Overlap Add first divides a given sample into a number of windows. Each of these windows overlaps for a certain amount with the other windows.

As you can see above, this roughly worked. The sample is stretched over time while the pitch is not affected much. However, the sample has become quite a bit louder and contains some artifacts. This is because the overlapping parts of the windows are being added together, resulting in a signal much higher than the original sample.

To counter this issue, the Overlap Add method cross-fades each of the overlapping windows. This is done by multiplying them by the Hann Function.
